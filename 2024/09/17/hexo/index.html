部署工具

  • mac系统电脑
  • 阿里云账号和域名 https://www.aliyun.com/
  • Github账号 https://github.com/
  • NewCDN网站 https://user.new-cdn.com/

基本要求

  1. 熟悉基本的命令行操作
  2. 注册相应账号并购买一个域名（我的几块钱一年）

基本流程

  1. 搭建hexo博客框架
  2. 推送到github仓库
  3. 配置CDN
  4. 博客的主题配置（自用matery：readme中有详细的配置教程可自行摸索）

搭建环境

  1. 官网傻瓜式安装Node.js http://nodejs.org/
  2. 安装Git http://sourceforge.net/projects/git-osx-installer/
  3. 安装cnpm镜像源（比较快）
    npm install -g cnpm --registry=http://registry.npm.taobao.org
  4. 命令行指令查看版本：检测是否但安装成功
    cnpm -v
    node -v
    git --version

部署Hexo

cnpm install -g hexo-cli(安装hexo框架)
hexo -v（查看版本检测是否安装成功）
mkdir blog（创建新文件夹并命名为blog）
cd blog（进入blog根目录）
sudo hexo init（初始化hexo）
hexo s（在本地生成hexo）
http://localhost:4000/（本地hexo网址）
hexo n "my first page"（新建一个文章名为my first page）
hexo cl（清除hexo缓存）
hexo g（生成hexo博客）
cnpm install --save hexo-deployer-git（装hexo推送插件)

注册github账号建并仓库

  1. 新建一个新的仓库
  2. 仓库名必须为：username.github.io（这里举例为ivwan.github.io方便后续配置对比）
  3. 仓库描述随便填
  4. 仓库状态设为公共public
  5. 创建仓库并保留SSH状态下的网址

配置GitHub账号信息

git config --global user.name "yourname"
git config --global user.email "youremail"

设置密钥

ssh-keygen -t rsa -C "youremail"(生成密钥/一路按回车)
vim id_rsa.pub(复制公钥到：github头像——setting——SSH and GPG keys——添加SSH公钥)或者cat ～/.ssh/id_rsa.pub（抓取公钥）
titile随便起个名，将从id_rsa.pub复制的key粘贴进keys栏并保存
配置完成后检测配置是否成功ssh -T git@github.com（返回yes就是链接成功）

配置hexo根目录下的_config.yml文件

deploy:
type: git
repo: 将新建仓库后的SSH网址粘贴至此
branch: master

(找到文件中此处并如上述填写，注意冒号后面有一个空格别丢了)

推送hexo到github仓库

hexo cl（清除hexo缓存）
hexo g（生成静态文章）
hexo d（推送到github仓库）

注册阿里云账号购买一个域名

假设购买的域名是ivwan.top（举例为了方便理解CDN配置）
要进行域名备案（需要较长时间建议提前进行）

注册获取一个CDN（自用NewCDN免费）与配置CDN

  1. 点击右侧接入域名——添加域名——套餐选默认的——加速域名可以写cdn.ivwan.top（前缀可自行填写，后续和购买域名一致）——取源协议默认HTTP——主源站网址写ivwan.github.io(写网站地址)——确认添加
  2. 添加完成后获取到一个系统分配的CNAME域名，将其复制并进入到阿里云的域名解析界面,选择已购买的域名选择添加记录——记录类型选择CNAME——主机记录写cdn（即刚才的加速域名前缀）——解析请求来源默认——记录值粘贴CDN分配的域名——TTL为10不变
  3. 配置完成稍等几分钟等代配置生效，可以点选生效测试按其跳转的网页描述看是否已经成功
  4. 配置完成后再次输入网站ivwan.github.io（输入你的自己的地址）访问发现速度会快很多

部署过程中遇到的问题

  1. 安装hexo插件很慢注意要用cnpm镜像源安装会很快，不要用npm指令
  2. 各类权限问题解决：Error: EACCES: permission denied
    • sudo su并输入电脑密码进入超级管理员模式再次执行
    • sudo chown -R [yourusername] [文件路径] 不行的话将文件路径会退到上一级再试试
    • 全局重装npm sudo npm install -g @vue/cli
    • 不要频繁启用管理员模式，不熟悉会对文件造成不可逆的损坏
  3. 推送代码到仓库时被远端服务器拒绝:
    个人头像——settings——Code security and analysis——push protection for yourself的状态改为关闭 并且在仓库的settings中的——push protetion也关闭(再此推送即可，推送完即可恢复保护设置)
  4. 检索网站时404 found not file :
    原因是博客不能一片文章都不留，用hexo n "1.md"指令新建一个文章即可
  5. 报错[Error: EACCES: permission denied, open ‘/Users/ivwan/blog/public/sw.js’] {
    errno: -13,
    code: ‘EACCES’,
    syscall: ‘open’,
    path: ‘/Users/ivwan/blog/public/sw.js’
    }
    解决:sudo chmod -R 777 /Users/ivwan/blog/public(还是权限问题)
  6. 报错Error: EACCES: permission denied, unlink ‘/Users/ivwan/blog/.deploy_git/about/index.html’ at async Object.unlink (node:internal/fs/promises:1059:10)
    解决:在超级管理员模式下再次操作即可（某文件unlink问题）
  7. Port 4000 has been used. Try other port instead.（4000端口已被占用）
    • 查询端口使用情况： Lsof -i:4000
    • 获取到端口使用信息其中有一个关键词PID
    • 杀死进程，释放端口:kill -9 PID号
      8.注意文中用到的ivwan为github的用户名，需要换成自己的用户名进行所有的操作，文中只是示例要注意替换
